2. 已修复的潜在问题
① 爬取页 doParseUrls 合并笔记时的状态陈旧
问题：用闭包里的 notes 做合并，快速连续解析时可能用旧状态，导致新解析结果被覆盖或重复。
修改：改为从 localStorage 读取当前笔记再合并；若读取失败（如数据损坏）则用内存中的 notes 兜底，并写入 localStorage 与 setNotes。
② 待办页 localStorage 解析异常
问题：JSON.parse(rawNotes)、JSON.parse(rawHistory) 在数据损坏时可能抛错，导致整段逻辑中断。
修改：对两处 JSON.parse 增加 try/catch，解析失败时使用空数组，避免崩溃。
